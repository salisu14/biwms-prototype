# Client Feedback Workflow

Use this template during the pilot so feedback is consistent, triageable, and easy to reproduce.

## Feedback Type

Choose one:

- Bug
- Enhancement
- Report request
- Workflow issue
- Training issue

## Priority

Choose one:

- Critical: blocks posting, security, month-end, payroll, inventory movement, or financial reporting.
- High: blocks an important workflow but has a temporary workaround.
- Medium: slows users down or causes confusion.
- Low: cosmetic, wording, training, or convenience request.

## Screenshot And Log Guidance

- Attach screenshots for UI issues, validation errors, report totals, and confusing workflows.
- Include the full page URL when possible.
- Include the document number, item code, customer/vendor code, payment number, journal batch, or production order number.
- Do not share passwords, MFA codes, recovery codes, tokens, or private customer bank details.
- For posting or report issues, also attach the relevant reconciliation command output where practical.

## Triage Notes

Internal triage should classify each item as:

- Accepted
- Needs more information
- Duplicate
- Training/documentation
- Deferred
- Not planned

For accepted issues, record the target release, owner, and test evidence before closing.
